# Public / Private Boundary

## Public

- Project name, purpose, and status.
- Public-safe screenshots and visual assets.
- Workflow diagrams that do not reveal implementation details.
- WordPress page draft.
- Ownership, security, and commercial-use policies.

## Private

- Source code.
- WordPress plugin/theme internals.
- Prompts and agent instructions.
- Credentials, tokens, cookies, and account dashboards.
- Client records and customer data.
- Payment, billing, legal/admin, medical, benefits, and family information.
- Deployment systems and operational infrastructure.
- Private receipts and verifier internals.

## Rule

Public surface. Private engine. Receipts always.
